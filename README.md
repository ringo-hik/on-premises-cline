# Cline On-Premises Edition

> On-premises version of Cline - 내부 네트워크 환경을 위한 독립적인 AI 코딩 어시스턴트

## 🔧 개발 접근 방식

이 프로젝트는 "오류 주도 개발" 방식을 채택하여 진행했습니다:

1. 실제로 인터넷 연결을 차단한 환경에서 Cline을 실행
2. 발생하는 모든 외부 연결 시도 및 오류 기록
3. 각 오류의 원인이 되는 코드 부분 식별 및 수정/제거
4. 수정된 버전에서 다시 테스트하며 반복

### 주요 개발 목표

**모든 외부 의존성 제거**
- API 호출, 텔레메트리, 인증 메커니즘 등 모든 외부 연결 요소 제거
- 필요한 리소스는 모두 로컬에 번들링
- 완전한 온프레미스 환경에서 작동 가능

## 🚀 주요 특징

### 🔐 내부 LLM Provider 지원

Internal LLM (OpenAI Compatible) 방식을 통해 내부망 모델과 연동합니다.

### 🛡️ 온프레미스 환경 최적화

#### 비활성화된 외부 연결 서비스

- **PostHog 텔레메트리**: Cline은 기본적으로 사용 통계, 오류 보고, 기능 사용 패턴을 PostHog 서비스로 전송합니다. 온프레미스 버전에서는 이 데이터 수집 기능이 완전히 제거되었습니다. 이를 통해 내부 코드 컨텍스트나 사용자 작업 데이터가 외부로 유출되는 위험을 차단합니다.

- **Firebase 인증**: 원본 Cline은 Google Firebase를 통한 사용자 인증을 사용합니다. 이 인증 시스템은 외부 서버에 의존하므로 온프레미스 버전에서는 완전히 제거되었습니다. 대신 로컬 실행 방식으로 인증 없이 작동합니다.

- **MCP 마켓플레이스**: Model Context Protocol(MCP) 마켓플레이스는 확장 도구를 다운로드하는 기능으로, 외부 서버 연결이 필요합니다. 온프레미스 버전에서는 이 기능이 비활성화되어 외부 도구 다운로드가 차단됩니다.

- **계정 로그인 UI**: Cline 계정 연동 UI 및 관련 백엔드 연결 요소가 제거되었습니다. 이를 통해 인증 과정에서 내부 정보가 외부로 전송될 가능성을 차단합니다.

- **자동 업데이트 확인**: VS Code 마켓플레이스 연결을 통한 자동 업데이트 확인 기능이 제거되었습니다.

이러한 변경으로 모든 데이터와 처리가 내부 네트워크 내에서만 이루어지도록 보장합니다.

## 📦 설치 가이드

### VS Code 마켓플레이스
```
1. VS Code 확장 프로그램 탭 열기
2. "Cline On-Premises" 검색
3. "Install" 클릭
```

### 수동 설치 (VSIX) - 온프레미스 환경 권장
```bash
# VSIX 파일을 내부망으로 전송 후
code --install-extension cline-internal-network.vsix
```

## ⚙️ 설정 방법

### Internal LLM 설정 (OpenAI Compatible)

1. **기본 설정**
   ```
   Settings → Cline → API Provider → OpenAI 선택
   ```

2. **엔드포인트 설정**
   ```
   Base URL: https://your-internal-llm.company.com/v1
   API Key: your-internal-api-key
   Model: gauss (내부망 모델)
   ```

> ⚠️ **중요**: 엔드포인트 URL에는 반드시 `/chat/completions`이 포함되지 않아야 합니다. 이는 자동으로 추가됩니다.

## 🔧 고급 설정

### 프록시 설정
내부 프록시를 통한 연결이 필요한 경우:
```json
{
  "http.proxy": "http://proxy.company.com:8080",
  "https.proxy": "http://proxy.company.com:8080",
  "http.proxyStrictSSL": false
}
```

### 타임아웃 설정
느린 내부 네트워크 환경을 위한 설정:
```json
{
  "cline.apiTimeout": 120000,
  "cline.retryAttempts": 5
}
```

### SSL 인증서 설정
자체 서명 인증서 사용 시:
```json
{
  "http.systemCertificates": false,
  "http.experimental.systemCertificatesV2": false
}
```

## 🔒 보안 고려사항

- 모든 API 키와 토큰은 VS Code 설정에 안전하게 저장됩니다
- 내부 네트워크에서만 작동하도록 설계되었습니다
- 외부로의 데이터 전송은 완전히 차단되어 있습니다
- 기업 보안 정책을 준수합니다
- 온프레미스 환경에서 완벽하게 작동합니다

## ⚠️ 알려진 이슈

- **UI 깨짐 현상**: Content Security Policy(CSP) 이슈로 버튼 아이콘들이 "ㅁ" 문자로 보이는 현상이 있습니다. 기능상 문제는 없으며 다음 방법으로 해결할 수 있습니다:
  - VS Code 설정(Ctrl+,)에서 "security.csp.enabled"를 검색하여 비활성화
  - 또는 설정 JSON에 다음 추가: `"workbench.webview.additionalCsp": "default-src 'self' https: data: 'unsafe-inline' 'unsafe-eval'; font-src 'self' https: data: *;"`

- **설정 저장 문제**: VS Code를 재시작하면 일부 설정이 초기화되는 문제가 있습니다.

- **특정 네트워크 환경**: 일부 제한적인 내부망 환경에서 연결 문제가 발생할 수 있습니다.

- **커스텀 작성 UI 이슈**: 최초 커스텀 작성 시 입력 칸이 이상하게 밀리면서 잘 안써지는 현상이 있습니다. 다른 곳에 작성 후 복사/붙여넣기로 입력하는 것을 권장합니다(작성 중인 헤더가 가장 밑으로 이동하지만 내용은 유지됩니다).

## 🌐 프로젝트 배경

이 프로젝트는 외부 인터넷 연결 없이 내부망(온프레미스 환경)에서 완전히 독립적으로 작동하는 Cline 버전을 개발하기 위해 시작되었습니다.

### 개발 동기

**사용자 본인이 사용하고 싶어서 시작했습니다.**

- 사내 온프레미스망 사용자들을 위한 AI 코딩 보조 도구 제공 계획
- 보안상의 이유로 외부 연결이 제한된 환경에서도 Cline의 기능 활용 희망

### 첫 번째 시도: 단순 API 변경의 한계

초기에는 API Provider만 내부망 모델로 변경하여 간단히 해결하려 했습니다. 하지만 다양한 외부 의존성 관련 에러가 발생했습니다:

- PostHog 및 기타 외부 분석 서비스 연동 시도
- MCP(Model Context Protocol) 관련 외부 연결
- Firebase 인증 작동 시도
- CDN을 통한 자원 로딩 시도
- VS Code 마켓플레이스 업데이트 확인

### 데이터 유출 우려로 프로젝트 보류

- 사용자가 실수로 잘못된 버튼을 클릭하여 내부 데이터가 외부로 유출될 위험
- 회사 보안 정책 위반 가능성
- 이러한 보안 우려로 인해 초기 계획을 잠시 보류

### 프로젝트 재개: 온프레미스 Cline 개발

이러한 문제를 근본적으로 해결하기 위해 온프레미스 Cline 개발을 개인 프로젝트로 재시작했습니다. 모든 외부 의존성을 제거하고 순수하게 내부망에서만 작동하는 버전 개발을 목표로 설정했습니다.
